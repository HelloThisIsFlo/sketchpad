4. Create `src/sketchpad/config.py` with a LAZY configuration pattern. This is critical -- config must NOT read env vars at module import time, because the server module imports config at the top level and we need imports to succeed without a .env file (e.g., for testing, Docker builds, CI).

   Define a `get_config()` function that reads env vars at call time and returns a dict (or a dataclass/namedtuple):
   ```python
   import os
   from functools import lru_cache

   @lru_cache(maxsize=1)
   def get_config():
       """Load configuration from environment. Called at server startup, not at import time."""
       return {
           # Required -- KeyError if missing (fail fast at startup, not import)
           "GITHUB_CLIENT_ID": os.environ["GITHUB_CLIENT_ID"],
           "GITHUB_CLIENT_SECRET": os.environ["GITHUB_CLIENT_SECRET"],
           "JWT_SIGNING_KEY": os.environ["JWT_SIGNING_KEY"],
           "STORAGE_ENCRYPTION_KEY": os.environ["STORAGE_ENCRYPTION_KEY"],
           # Optional with defaults
           "SERVER_URL": os.environ.get("SERVER_URL", "http://localhost:8000"),
           "DATA_DIR": os.environ.get("DATA_DIR", "./data"),
           "STATE_DIR": os.environ.get("STATE_DIR", "./state"),
           "SKETCHPAD_FILENAME": os.environ.get("SKETCHPAD_FILENAME", "sketchpad.md"),
           "SIZE_LIMIT": int(os.environ.get("SIZE_LIMIT", "50000")),
       }
   ```

   The @lru_cache ensures env vars are read exactly once (at first call), not on every access. Downstream code calls `get_config()["KEY"]` instead of importing bare constants.

   Do NOT use bare module-level `GITHUB_CLIENT_ID = os.environ["GITHUB_CLIENT_ID"]` -- that pattern causes KeyError on import when no .env is present.

5. Create `.env.example` with placeholder values for all env vars. Include comments explaining each var, how to generate keys (JWT_SIGNING_KEY via `python -c "import secrets; print(secrets.token_urlsafe(32))"`, STORAGE_ENCRYPTION_KEY via `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`), and a note about updating SERVER_URL with cloudflared tunnel URL each dev session.

6. Update `.gitignore` (create if not exists): add `.env`, `.venv/`, `__pycache__/`, `*.pyc`, `data/`, `state/`, `uv.lock` is NOT gitignored (it should be committed per uv best practices).

Note: Do NOT add python-dotenv as a dependency -- FastMCP auto-loads .env files on import.

<task type="auto">
  <name>Task 2: Create FastMCP server with GitHubProvider and file tools</name>
  <files>src/sketchpad/server.py, src/sketchpad/tools.py, src/sketchpad/__main__.py</files>
  <action>
1. Create `src/sketchpad/server.py`:
   - Import get_config from sketchpad.config (NOT bare constants)
   - Import register_tools from sketchpad.tools
   - Define a `create_app()` factory function that:
     - Calls `cfg = get_config()` to read env vars (this is where KeyError fires if vars are missing)
     - Sets up FileTreeStore with FernetEncryptionWrapper for OAuth state persistence:
       ```python
       from key_value.aio.stores.filetree import (
           FileTreeStore,
           FileTreeV1KeySanitizationStrategy,
           FileTreeV1CollectionSanitizationStrategy,
       )
       from key_value.aio.wrappers.encryption import FernetEncryptionWrapper
       from cryptography.fernet import Fernet
       ```
     - Creates state_dir Path from cfg["STATE_DIR"], mkdir(parents=True, exist_ok=True)
     - Instantiates FileTreeStore with state_dir and both sanitization strategies
     - Wraps in FernetEncryptionWrapper using Fernet(cfg["STORAGE_ENCRYPTION_KEY"])
     - Creates GitHubProvider with cfg["GITHUB_CLIENT_ID"], cfg["GITHUB_CLIENT_SECRET"], base_url=cfg["SERVER_URL"], jwt_signing_key=cfg["JWT_SIGNING_KEY"], client_storage=encrypted_store
     - Creates FastMCP instance: `mcp = FastMCP(name="Sketchpad", auth=auth)`
     - Calls register_tools(mcp) to register tools
     - Returns mcp

   - Do NOT create the app at module level. The module should only define create_app() and imports. This ensures `import sketchpad.server` always succeeds regardless of env var state.

   - Add `if __name__ == "__main__"` block:
     ```python
     if __name__ == "__main__":
         app = create_app()
         app.run(transport="http", host="0.0.0.0", port=8000)
     ```

2. Create `src/sketchpad/tools.py`:
   - Import os, Path from pathlib
   - Import get_config from sketchpad.config
   - Define WELCOME_MESSAGE = "Welcome to Sketchpad! Write something here."
   - Define a `register_tools(mcp)` function that registers both tools on the passed mcp instance:

   `read_file()` tool:
   - Docstring: "Read the sketchpad file. This is a single shared Markdown file for jotting down notes, ideas, and drafts."
   - Call cfg = get_config() inside the tool function to get DATA_DIR, SKETCHPAD_FILENAME, SIZE_LIMIT
   - Compute SKETCHPAD_PATH = Path(cfg["DATA_DIR"]) / cfg["SKETCHPAD_FILENAME"]
   - If file doesn't exist, return WELCOME_MESSAGE
   - Read file content (utf-8)
   - If len(content) > cfg["SIZE_LIMIT"], append warning: "\n\n---\n[WARNING: File exceeds recommended size limit (~50KB). Consider trimming older content.]"
   - Return content

   `write_file(content: str, mode: str = "replace")` tool:
   - Docstring: "Write to the sketchpad file. Use this for notes, drafts, ideas -- Markdown formatting recommended but not required.\n\nArgs:\n    content: The text to write.\n    mode: 'replace' (default) overwrites the file; 'append' adds to the end."
   - Call cfg = get_config() inside the tool function
   - Compute SKETCHPAD_PATH = Path(cfg["DATA_DIR"]) / cfg["SKETCHPAD_FILENAME"]
   - Create parent dirs if needed: SKETCHPAD_PATH.parent.mkdir(parents=True, exist_ok=True)
   - If mode == "append": read existing content (or ""), concatenate, write
   - Else (replace): write content directly
   - Return confirmation: f"File updated ({mode} mode). Size: {SKETCHPAD_PATH.stat().st_size} bytes."

3. Create `src/sketchpad/__main__.py`:
   - This enables `python -m sketchpad`:
     ```python
     from sketchpad.server import create_app

     app = create_app()
     app.run(transport="http", host="0.0.0.0", port=8000)
     ```

Important: Use `@mcp.tool` decorator inside the `register_tools` function (tools are closures registered on the passed mcp instance). The RESEARCH.md shows `register_tools(mcp)` as the pattern.

Important: Since get_config() uses @lru_cache, calling it from multiple tool invocations is cheap (cached after first call). The tools read config at call time, not import time.
