<task type="auto">
  <name>Task 1: Set up .env and start server with cloudflared tunnel</name>
  <files>.env</files>
  <action>
1. Create `.env` from `.env.example` if it doesn't already exist. The user's GitHub OAuth credentials should already be available as K8s secrets from Phase 1. Help the user populate the .env:
   - GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET: retrieve from `kubectl get secret github-oauth -n sketchpad -o jsonpath='{.data.client-id}' | base64 -d` and similar for client-secret
   - JWT_SIGNING_KEY: generate with `python -c "import secrets; print(secrets.token_urlsafe(32))"`
   - STORAGE_ENCRYPTION_KEY: generate with `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`
   - SERVER_URL: will be set after cloudflared starts

2. Start cloudflared ephemeral tunnel in background:
   ```bash
   cloudflared tunnel --url http://localhost:8000
   ```
   Capture the random *.trycloudflare.com URL from the output.

3. Update .env with the tunnel URL as SERVER_URL.

4. Update the GitHub OAuth App callback URL to `{tunnel_url}/auth/callback`. Print instructions for the user to do this in GitHub Settings > Developer settings > OAuth Apps.

5. Start the server:
   ```bash
   uv run python -m sketchpad
   ```
   Verify it starts without errors and binds to 0.0.0.0:8000.

6. Quick smoke test:
   ```bash
   curl -sf {tunnel_url}/.well-known/oauth-authorization-server | jq .
   ```
  </action>
  <verify>
    <automated>curl -sf http://localhost:8000/.well-known/oauth-authorization-server | jq -e '.authorization_endpoint' > /dev/null && echo "PASS"</automated>
  </verify>
  <done>Server running locally, cloudflared tunnel active, discovery endpoint returns valid JSON with correct SERVER_URL</done>
</task>
