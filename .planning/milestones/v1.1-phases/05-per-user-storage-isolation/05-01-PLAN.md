Create `src/sketchpad/user_identity.py` implementing the `resolve_user_dir` function:

```python
import re
import logging
from pathlib import Path

logger = logging.getLogger(__name__)

# GitHub: alphanumeric + hyphens, 1-39 chars, no leading/trailing hyphens
GITHUB_USERNAME_RE = re.compile(r"^[a-z0-9]([a-z0-9-]*[a-z0-9])?$")


def resolve_user_dir(data_dir: str, provider: str, raw_identifier: str) -> Path:
    """Resolve a user's data directory with sanitization and traversal defense.

    Args:
        data_dir: Base data directory path (e.g., "/data").
        provider: OAuth provider name (e.g., "github").
        raw_identifier: Raw username/identifier from OAuth token.

    Returns:
        Resolved Path to the user's data directory.

    Raises:
        ValueError: If the identifier is invalid or path traversal is detected.
    """
```

Implementation details:
1. If `provider == "github"`:
   - Lowercase the identifier: `identifier = raw_identifier.lower()`
   - Validate against `GITHUB_USERNAME_RE` and `len(identifier) <= 39`
   - If invalid: log at WARNING "Invalid GitHub username attempted: %s" with raw_identifier, raise `ValueError("Invalid request")`
2. If provider is unknown: raise `ValueError(f"Unknown provider: {provider}")`
3. Construct path: `base = Path(data_dir).resolve()`, `user_dir = (base / provider / identifier).resolve()`
4. Defense-in-depth: if `not user_dir.is_relative_to(base)`: log WARNING "Path traversal attempt: %s" with raw_identifier, raise `ValueError("Invalid request")`
5. Return `user_dir`

Note: This function does NOT create directories. Directory creation happens in tools.py on write. This keeps the function pure and testable.
