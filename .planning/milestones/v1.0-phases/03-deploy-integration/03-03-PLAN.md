Layer-by-layer deployment and verification (per CONTEXT.md locked decision). This task produces no new files -- it operates on the cluster using artifacts from Plans 01 and 02.

**Pre-flight: Update GitHub OAuth App callback URL**
Before deploying, the user must update the GitHub OAuth App callback URL to production. Check if this needs to be done by noting it -- the checkpoint in Task 3 will confirm.

The correct callback URL is: `https://sketchpad.kempenich.ai/auth/callback`
(NOT /github/callback -- see Phase 1 doc fix in Plan 02)

**Layer 1: Deploy to K8s**
1. Delete the placeholder: `kubectl delete deployment sketchpad-placeholder -n sketchpad` and `kubectl delete configmap placeholder-config -n sketchpad`
2. Apply new manifests: `kubectl apply -f k8s/deployment.yaml -f k8s/service.yaml -n sketchpad` (or `make deploy` if image is already pushed)
3. Wait: `kubectl rollout status deployment/sketchpad -n sketchpad --timeout=120s`
4. Verify: `kubectl get pods -n sketchpad` -- should show `sketchpad-xxxxx` Running

If the latest image is not on ghcr.io (CI hasn't run yet for the new code from Plans 01-02), run `make all` to build, push, and deploy in one step. Otherwise `make deploy` suffices since CI already pushed the image.

**Layer 2: Tunnel Connectivity**
5. Verify health: `curl -sf https://sketchpad.kempenich.ai/health` -- should return `{"status":"ok","service":"sketchpad"}`
6. Verify OAuth discovery: `curl -sf https://sketchpad.kempenich.ai/.well-known/oauth-authorization-server | python3 -m json.tool` -- should return JSON with authorization_endpoint, token_endpoint, registration_endpoint

If Layer 2 fails: check pod logs (`kubectl logs -f deployment/sketchpad -n sketchpad`), Service port mapping (`kubectl describe svc sketchpad -n sketchpad`), and cloudflared logs.

**Layer 3: OAuth Flow (automated)**
7. Run test_oauth.py against the live URL:
```bash
SERVER_URL=https://sketchpad.kempenich.ai uv run python test_oauth.py
```
Or update `.env` to set `SERVER_URL=https://sketchpad.kempenich.ai` and run `uv run python test_oauth.py`.

Note: test_oauth.py reads `.env` via dotenv_values(), so the env var override approach requires a small check. If it doesn't work, update .env directly.

8. Verify: Steps 1-3 of test_oauth.py (discovery, 401 challenge, DCR) should PASS. Steps 4-7 (browser OAuth) will need manual interaction.

**Layer 4: Persistence Test**
9. After successful tool calls (from Task 3 checkpoint), restart the pod: `kubectl rollout restart deployment/sketchpad -n sketchpad`
10. Wait for new pod: `kubectl rollout status deployment/sketchpad -n sketchpad`
11. Verify data survived: use Claude Code to read the sketchpad -- content from before restart should still be there
