  <files>src/sketchpad/middleware.py, src/sketchpad/config.py, src/sketchpad/__main__.py</files>
  <action>
1. Create `src/sketchpad/middleware.py` with `OriginValidationMiddleware` class:
   - Subclass `starlette.middleware.base.BaseHTTPMiddleware`
   - Constructor takes `app`, `allowed_origins: list[str]`, `protected_paths: list[str] | None = None` (default `["/mcp"]`)
   - Store allowed_origins as a `set` for O(1) lookup, protected_paths as a `set`
   - `dispatch()` method:
     - If `request.url.path` not in protected_paths, pass through (`call_next`)
     - If no Origin header present, pass through (non-browser clients like curl/CLI)
     - If Origin in allowed_origins set, pass through
     - Otherwise: log warning with origin, client IP, and path; return 403 JSONResponse with `{"error": "origin_not_allowed", "detail": "Origin '{origin}' is not in the allowlist"}`
   - Use `logging.getLogger(__name__)` for the logger

2. Update `src/sketchpad/config.py` — add `ALLOWED_ORIGINS` to the dict returned by `get_config()`:
   ```python
   "ALLOWED_ORIGINS": [
       o.strip() for o in
       os.environ.get("ALLOWED_ORIGINS", "https://claude.ai,https://www.claude.ai").split(",")
       if o.strip()
   ],
   ```
   Place it in the optional-with-defaults section (after SERVER_URL, before provider-specific).

3. Update `src/sketchpad/__main__.py` to wire middleware:
   - Add imports: `from starlette.middleware import Middleware` and `from sketchpad.middleware import OriginValidationMiddleware` and `from sketchpad.config import get_config`
   - After `app = create_app()`, get config and build middleware list:
     ```python
     cfg = get_config()
     middleware = [
         Middleware(
             OriginValidationMiddleware,
             allowed_origins=cfg["ALLOWED_ORIGINS"],
         ),
     ]
     ```
   - Update `app.run()` call to pass middleware: `app.run(transport="http", host="0.0.0.0", port=8000, middleware=middleware)`

Do NOT add CORS middleware (user decision: no CORS). Do NOT add Origin checks to /authorize or /token (research recommendation: these are OAuth flow endpoints, adding Origin checks risks breaking the flow). Do NOT add redundant token validation middleware (SEC-02 is verification-only per user decision).
