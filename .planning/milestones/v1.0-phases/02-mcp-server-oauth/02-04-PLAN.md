Root cause analysis (from plan-phase investigation):

The FastMCP OAuthProvider.get_routes() at line 675 calls self.set_mcp_path(mcp_path) where mcp_path="/mcp".
This sets self._resource_url to "{base_url}/mcp" (e.g., http://localhost:8000/mcp).
At line 721-733, it calls create_protected_resource_routes(resource_url=self._resource_url, ...).
In mcp/server/auth/routes.py:243, build_resource_metadata_url() constructs the path as:
  /.well-known/oauth-protected-resource{resource_path}
where resource_path is /mcp, giving /.well-known/oauth-protected-resource/mcp.

The endpoint IS being served. The test was requesting the wrong URL.
</context>
