   - FROM python:3.12-slim

<task type="auto">
  <name>Task 2: Create test-oauth.sh and MCP Inspector guide</name>
  <files>test-oauth.sh, docs/mcp-inspector.md</files>
  <action>
1. Create `test-oauth.sh` -- the end-to-end OAuth flow test script. This is the PRIMARY test client per user decision. Structure:

   Header: #!/usr/bin/env bash, set -euo pipefail, SERVER="${1:-http://localhost:8000}"

   Step 1 - Discovery (automated):
   - Fetch /.well-known/oauth-authorization-server, pipe through jq, verify authorization_endpoint and token_endpoint and registration_endpoint are present
   - Fetch /.well-known/oauth-protected-resource, pipe through jq, verify resource field present

   Step 2 - Unauthenticated 401 (automated):
   - POST /mcp without token, capture HTTP status code, assert 401
   - Show WWW-Authenticate header from response

   Step 3 - Dynamic Client Registration (automated):
   - POST /register with JSON body: client_name "test-oauth-script", redirect_uris ["http://localhost:9999/callback"], grant_types ["authorization_code", "refresh_token"], response_types ["code"], token_endpoint_auth_method "none"
   - Extract client_id from response

   Step 4 - Authorization (manual -- requires browser):
   - Generate PKCE code_verifier (openssl rand -base64 32, sanitize for URL-safe)
   - Generate code_challenge (SHA-256 of code_verifier, base64url-encode)
   - Generate state (openssl rand -hex 16)
   - Construct authorize URL with all params
   - Print URL for user to open in browser
   - After GitHub login and redirect, user pastes the authorization code from the redirect URL
   - Use `read -r AUTH_CODE` to capture

   Step 5 - Token Exchange (automated after user input):
   - POST /token with grant_type=authorization_code, code, client_id, redirect_uri, code_verifier
   - Extract access_token and refresh_token
   - Check expires_in is present

   Step 6 - Refresh Token (automated):
   - POST /token with grant_type=refresh_token, refresh_token, client_id
   - Extract new access_token

   Step 7 - MCP Tool Calls (automated):
   - POST /mcp with Authorization: Bearer header, JSON-RPC tools/list request
   - POST /mcp with tools/call for read_file
   - POST /mcp with tools/call for write_file (content: "Hello from test-oauth.sh!")
   - POST /mcp with tools/call for read_file again (verify written content)

   Each step should:
   - Print a clear "=== Step N: Description ===" header
   - Show the curl command output formatted with jq
   - Print PASS/FAIL for each assertion
   - Exit with non-zero if any step fails

   Make the script executable: chmod +x test-oauth.sh

   IMPORTANT: The MCP JSON-RPC calls may need an `initialize` handshake first before tools/list. Include an initialize request (method: "initialize", params: {protocolVersion: "2025-03-26", capabilities: {}, clientInfo: {name: "test-oauth-script", version: "1.0.0"}}) before tools/list. The MCP Streamable HTTP transport may require session handling (check for Mcp-Session-Id header in initialize response and include it in subsequent requests).

2. Create `docs/mcp-inspector.md` -- MCP Inspector "fun things to try" guide per user decision. Frame as a learning opportunity, not test steps:
   - Brief intro: what MCP Inspector is and why it's useful
   - Installation: `npx @anthropic/mcp-inspector` (or however it's currently installed)
   - How to connect to the local server (URL, transport type)
   - "Fun things to try" section with numbered explorations:
     1. Browse the tools list -- see read_file and write_file with their descriptions
     2. Call read_file with no args -- see the welcome message
     3. Write something -- call write_file with some text
     4. Read it back -- verify your text appears
     5. Try append mode -- write_file with mode: "append"
     6. Check the OAuth endpoints -- look at the discovery metadata
   - Note that MCP Inspector handles OAuth automatically (it will prompt for login)
   - Keep it concise and friendly -- this is a "play around and learn" guide
