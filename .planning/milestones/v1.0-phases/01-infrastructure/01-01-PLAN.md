- Title: "Creating the GitHub OAuth App"
- Prerequisites: GitHub account
- Steps:

  2. Click "New OAuth App"
  3. Fill in fields:
     - Application name: `Sketchpad` (or any name)
     - Homepage URL: `https://sketchpad.kempenich.ai`
     - Authorization callback URL: `https://sketchpad.kempenich.ai/github/callback`
  4. Click "Register application"
  5. Copy the Client ID (shown on the app page)
  6. Click "Generate a new client secret"
  7. Copy the Client Secret immediately (shown only once)

- Section: "Creating the Kubernetes Secret" -- reference the command from k8s/secrets/README.md with placeholders
- Note: The callback URL must match exactly what the FastMCP server expects (Phase 2 will configure this)

**docs/synology-nfs.md:**
- Title: "Setting Up NFS on Synology for Kubernetes"
- Target: DSM 7.2
- Prerequisites: Admin access to Synology DSM
- Steps:
  1. Enable NFS: Control Panel > File Services > NFS tab > Enable NFS service, set maximum NFS protocol to NFSv4.1
     - Verification: "NFS service status should show 'Running'"
  2. Create shared folder: Control Panel > Shared Folder > Create
     - Name: `k8s` (path will be `/volume1/k8s`)
     - Do NOT enable data checksum (unnecessary overhead for K8s workloads)
     - Verification: "Shared folder 'k8s' appears in the list"
  3. Set NFS permissions: Select `k8s` folder > Edit > NFS Permissions tab > Create
     - Hostname or IP: `192.168.0.0/24` (covers all cluster nodes on same subnet)
     - Privilege: Read/Write
     - Squash: "Map all users to admin" (critical -- avoids permission denied errors with K8s pods)
     - Security: `sys`
     - Enable asynchronous: Yes (better performance)
     - Verification: "NFS rule appears in the permission list showing the subnet"
  4. Verify NFS export is visible: Run `showmount -e 192.168.0.102` from local machine -- should show `/volume1/k8s`
  5. Add to Hyper Backup: Open Hyper Backup app > Edit existing backup task (or create new) > Add `/volume1/k8s` to the backup selection
     - Verification: "The k8s folder appears in the backup task's folder selection"
- Warning box: "The squash setting 'Map all users to admin' is required. Without it, K8s pods will get 'Permission denied' errors because they run as UIDs that don't exist on the Synology."

**docs/cloudflare-tunnel.md** (DOCS-03):
- Title: "Setting Up Cloudflare Tunnel for Sketchpad"
- Prerequisites: Cloudflare account with `kempenich.dev` domain
- Steps:
  1. Go to Cloudflare Zero Trust dashboard > Networks > Tunnels (exact URL: https://one.dash.cloudflare.com/ then navigate)
  2. Click "Create a tunnel"
  3. Select "Cloudflared" as the connector
  4. Name the tunnel: `sketchpad`
  5. On the "Install and run connectors" page: copy the tunnel token (the long string after `--token` in the install command). Do NOT install -- K8s handles this.
     - Verification: "You have a token string starting with `ey...`"
  6. Click Next to configure the public hostname:
     - Subdomain: `thehome-sketchpad`
     - Domain: `kempenich.dev`
     - Service type: HTTP
     - Service URL: `sketchpad.sketchpad.svc.cluster.local:80`
  7. Save the tunnel
     - Verification: "Tunnel shows as 'Inactive' (expected -- cloudflared pod not deployed yet)"
  8. Create the Kubernetes Secret with the tunnel token -- reference command from k8s/secrets/README.md
  9. After deploying cloudflared (Plan 02), tunnel should show "Healthy"
- Config reference: The service URL format is `<service-name>.<namespace>.svc.cluster.local:<port>`. The Service `sketchpad` in namespace `sketchpad` on port 80 becomes `sketchpad.sketchpad.svc.cluster.local:80`.
